## 🔧 Что было исправлено

### 1. Backend DELETE Endpoint (УЛУЧШЕН)

**Файл:** `backend/src/routes/monolith-projects.js`

**Проблема:**
- Удаление только из `monolith_projects`
- Позиции привязаны к таблице `bridges` через FK
- Позиции оставались сиротами в БД

**Решение:**
```javascript
// Удаляем в правильном порядке:
1. DELETE FROM bridges (CASCADE → удалит positions автоматически)
2. DELETE FROM monolith_projects (CASCADE → удалит parts автоматически)
3. Verify: проверка что positions действительно удалены
```

### 2. Frontend Auto-Refresh (УЖЕ РАБОТАЛ)

**Файл:** `frontend/src/hooks/useBridges.ts`

**Механизм:**
- React Query автоматически обновляет UI через `invalidateQueries`
- После `deleteBridge()` или `deleteProject()` список обновляется
- Никакой ручной перезагрузки не требуется ✅

### 3. Два типа удаления

#### A. Удаление одного объекта (bridge)
```
DELETE /api/monolith-projects/:id
```
- Удаляет один объект из проекта
- Пример: удалить "SO 11-20-01" из проекта

#### B. Удаление всего проекта (folder)
```
DELETE /api/monolith-projects/by-project-name/:projectName
```
- Удаляет ВСЕ объекты с одинаковым project_name
- Пример: удалить весь проект "Nýřanz - Heřmanová Hut" (4 моста)

## 🐛 Что проверить если что-то не работает

### Проблема 1: Объект не исчезает из UI

**Симптомы:**
- Нажал удалить → модалка подтвердила → но объект всё ещё виден

**Диагностика:**
1. Открой консоль браузера (F12 → Console)
2. Найди запрос: `DELETE /api/monolith-projects/...`
3. Посмотри статус: должен быть `200 OK`

**Возможные причины:**
- Ошибка 404 → объект не найден в БД (не мигрирован?)
- Ошибка 500 → ошибка на бэкенде (смотри логи Render)
- Запрос не отправился → проблема в frontend коде

**Решение:**
- Если 404: проверь что Migration 005 выполнилась (смотри логи)
- Если 500: покажи мне логи бэкенда
- Если запрос не отправился: очисти кэш браузера (Ctrl+Shift+Delete)

---

### Проблема 2: Позиции не удаляются

**Симптомы:**
- Объект удалён из UI
- Но `positions.count` в debug endpoint не уменьшился

**Диагностика:**
1. Render Dashboard → Monolit-Planner API → Logs
2. Ищи: `[DELETE PROJECT] Will delete X positions`
3. Потом: `[DELETE PROJECT] ⚠️ X positions still remain!`

**Возможные причины:**
- FK constraint не работает (CASCADE не сработал)
- Позиции привязаны к другому bridge_id

**Решение:**
- Покажи мне логи бэкенда
- Проверь FK constraint в schema-postgres.sql

---

### Проблема 3: Проект не исчезает после F5

**Симптомы:**
- Удалил объект → исчез из UI
- Нажал F5 → объект **вернулся**!

**Диагностика:**
- Объект НЕ удалён из БД (только из frontend кэша)

**Возможные причины:**
- DELETE запрос вернул 200 но ничего не удалил
- Миграция 005 не выполнилась → данные в старой таблице bridges

**Решение:**
1. Проверь debug endpoint: объект всё ещё в `bridges` или `monolith_projects`?
2. Проверь логи бэкенда: есть ли строка `[DELETE PROJECT] ✓ Deleted...`?
3. Если объект в старой таблице `bridges` но не в `monolith_projects`:
   - Миграция 005 не выполнилась
   - Проверь логи: `[Migration 005] ✅ Successfully migrated...`

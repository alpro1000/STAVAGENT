ТЕКУЩИЙ FLOW
═══════════════════════════════════════════════════════════════════════════════

1. User clicks "💾 Nahrát XLSX"
2. File uploaded → uploadAPI.uploadXLSX(file)
3. Backend parses XLSX → creates bridges in database
4. Frontend updates:
   - setBridges() → adds to context
   - setSelectedBridge() → selects first bridge
   - Sidebar shows new bridges
5. Table shows positions

ПРОБЛЕМА: Нет связи с Portal!

РЕШЕНИЕ 1: ДОБАВИТЬ PORTAL LINK ПОСЛЕ ИМПОРТА (МИНИМАЛЬНОЕ)
═══════════════════════════════════════════════════════════════════════════════

После успешного импорта XLSX показать диалог:

┌─────────────────────────────────────────────────────────────────────────┐
│ ✅ Import úspěšný! Nalezeno 3 objektů s 45 pozicemi.                   │
│                                                                          │
│ Chcete vytvořit projekt v Portal?                                       │
│                                                                          │
│ [Ano, vytvořit projekt]  [Ne, pokračovat bez Portal]                   │
└─────────────────────────────────────────────────────────────────────────┘

IF "Ano":
1. Call Portal API: POST /api/portal-projects
   Body: { project_name, project_type: "custom", description }
2. Get portal_project_id from response
3. Call Monolit API: POST /api/monolith-projects/:id/link-portal
   Body: { portal_project_id }
4. Create kiosk link in Portal: POST /api/kiosk-links
   Body: { portal_project_id, kiosk_type: "monolit", kiosk_project_id }

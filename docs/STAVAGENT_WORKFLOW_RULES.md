# ПРАВИЛА РАБОТЫ — STAVAGENT (dev-воркфлоу + инварианты)

Дистиллят дисциплины, выработанной на W3 → W3b → detect/export.
Дом: добавить в `CLAUDE.md` (его читает агент) или держать как `docs/WORKFLOW.md`.

---

## 1. Контракт-first
Каждый воркстрим начинается с `docs/tasks/TASK_*.md`:
- цель; правила/инварианты; **kritéria приёмки** (продолжать глобальный счётчик #NN);
  границы (что НЕ трогать); поступ.
- Поведение описывать **plain prose, без имён переменных** — конкретные имена в коде
  выводятся из решерше репо, не задаются в спеке.
- Уникальное имя файла (не пересекаться с существующими TASK_*).

## 2. Ритм (не отклоняться)
1. **Рекогносцировка first** — прочитать весь репо, найти существующие структуры,
   **ничего не менять**, STOP на ревью. (Мантра: сначала читаешь весь репо → najdi
   existing structures → potom píšeš.)
2. Ревью карты → апрув + решения по открытым вопросам.
3. **RED-тесты first** — воспроизводят дефекты/kritéria; убедиться, что красные.
4. Имплементация → **GREEN** + регресс-гарды (прежние kritéria остаются зелёными).
5. CI — тесты **skip-proof + fail-loud**, оба `paths:`-блока + инвокация.
6. **Draft PR** → подтвердить, что CI зелёный **И тесты реально выполняются** (не skip).
7. Человек смотрит дифф → **человек мержит** (main — рукой, контроль за тобой).
- **Никогда не мержить на красном.** CI, ловящий регресс до мержа, — это страховка,
  а не помеха.

## 3. Инварианты
- **Детерминизм-first.** LLM только где реально нужно. LLM **планирует и решает**
  (какую тулзу, ветвиться ли, спросить ли человека), но **не считает числа и не
  выдумывает коды**.
- **Grounding.** `_source` на каждой произведённой позиции; никаких выдуманных
  данных/кодов (P26/P29). Provenance — в метаданные/audit-лист, **не ломая** канонический
  формат дельиверабла (KROS-лист чистый).
- **Тесты skip-proof + fail-loud.** Sync/`asyncio.run` без тихого skip; ошибка падает,
  а не маскируется (урок false-green async и lambda-регресса).
- **Один воркстрим = одна ветка.** Не запутывать in-flight PR другой работой.
- **Границы.** Не трогать `provider_router` и несвязанные прод-пути. Дрейф (TS-двойник
  `element-classifier.ts`) — **отмечать, не форкать логику**.
- **Источник правды классификации — Python-MCP.** Калькулятор и фронт потребляют;
  логику классов/армирования в двух местах не дублировать.
- **MCP-тулзы.** Одна ответственность, схема, без бизнес-логики/UI; регистрация в
  `server.py` + REST-обёртка + manifest + CI. Много мелких композируемых > пара больших.
- **calc-API по форме не менять без флага** — его парсит `calculate_concrete_works`.
- **Фронт vs upstream.** «Фронт выводит неверно» сначала разобрать: бэкенд прислал
  правильное и фронт криво нарисовал → чинить фронт; бэкенд прислал кривое без
  контекста → это трек оркестратора, фронт-заплаткой не маскировать.

## 4. Мерж-гейт
Перед мержем в main: CI зелёный + подтверждено, что тесты выполняются (не skip) +
дифф просмотрен человеком. Мержит человек.

## 5. Порядок крупных вех (актуально)
detect/export (merge) → **ADK-спайк** (решить оркестратор) → **искатель полей / extract**
(сначала поля из текста TZ, потом výměry из чертежей) → полный автономный путь
«сырой PDF → Excel».

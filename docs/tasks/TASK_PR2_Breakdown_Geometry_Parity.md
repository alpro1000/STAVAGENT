## 1. Контекст

Пин-тест #1514 зафиксировал (не починил) три класса завышений в
`create_work_breakdown`:

| эвристика | дефект | масштаб |
|---|---|---|
| soffit: area = V / 0,25 | дефолт толщины 0,25 м вместо реальной | ×1,8 на стропе 450 мм; ×2 на плите 0,5 м |
| стены: V / 0,3 × 2 | «стена 0,3 м» для фундаментных блоков | ~×7 от TS-периметра |
| подошва скружи: V / 0,25 | та же толщина-фикция | ×2–4 при реальных 0,5–1,0 м |

Классы риска: ÚRS/RTS-ветка эмитит опалубку построчно → завышения идут прямо
в офферту приватных проектов (hk212, RD Jáchymov). OTSKP прикрыт bundling'ом,
но скруж-дыра (`CATALOG_BUNDLING`: skruž не бандлится) открыта и там.

Канон количества уже существует: TS-движок (element-classifier/planner) считает
опалубку из геометрии профиля. Два источника истины = запрещённое состояние.

## 2. Целевое поведение

### 2.1 Один источник геометрической истины

Количества опалубки/скружи в breakdown выводятся ТЕМ ЖЕ правилом, что в TS-движке
(§4-parity прецедент: prismatic V=L·W·H, area=2(L+W)·H уже living в
calculate_concrete_works). Не «похожая формула в Python» — а формула,
запиненная parity-тестом против TS-выхода на одинаковых входах. Где формулу
нельзя выровнять без входов — см. 2.2, не выдумывать.

### 2.2 Лестница источников количества (per-item, видимая)

1. **explicit** — вызывающий дал area_m2/length_m×height → берём как есть, conf 1.0
2. **derived** — есть достаточная геометрия (толщина/высота/длина по типу) →
   детерминированная формула, quantity_status='computed', формула в quantity_formula
3. **absent** → **NEPOČÍTÁNO с именем недостающего входа** (образец тубуса:
   «chybí explicitní geometrie — вход X»). НИКОГДА дефолт-толщина.

Дефолты 0,25/0,3 умирают. Это поведенческий сдвиг: часть сегодняшних ответов
с цифрой станет NEPOČÍTÁNO. Это ЦЕЛЬ, не регрессия — уверенная чушь хуже
честной дыры (прецедент: тоннель за 8,3M).

### 2.3 Скруж

Подошва скружи: только из явной толщины/высоты конструкции (правило
TUBUS_SUPPORT_RULE — образец data-правила с источником). Плюс закрыть
bundling-дыру skruž из recon Q2 (`CATALOG_BUNDLING`): на OTSKP скруж
mostovky бандлится в NK-положку по TKP/спецификации — проверить по
реальной спецификации 421/422-семьи и запинить с цитатой (метод #1519:
правило читается из каталога). Если спецификация НЕ бандлит — не бандлить,
записать факт. Export last-mile дыра — туда же.

### 2.4 Что НЕ входит

- Grandfather-лист OTSKP-голов (`otskp-binding-fallback-heads`) — соседний тикет.
- Normalizer-таблица — соседний тикет.
- Multi-DC поток (объединение заливок, бригады на объект — Excel Александра) —
  отдельная спека PR2.5 после этого PR.
- Тубус: уже на §2.10-пути, не трогать (кроме shared-хелперов).

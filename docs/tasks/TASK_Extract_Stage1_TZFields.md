## 1. Что извлекаем (стадия 1)
Из технической зправы — **авторитетные поля**, нужные нижестоящим тулзам:
- **название объекта** и **строка характеристики** → кормят `detect_object_type`;
- **список конструкционных элементов** (именованные: NK, dříky pilířů, opěry, úložné prahy,
  římsy, základy, pilota…);
- **класс бетона на каждый элемент** (привязанный к элементу, не «висящий»).

Каждое поле несёт `_source` (страница/раздел) и confidence; где неуверенно → OVĚŘIT/null.

## 2. Ключевое правило (та же ловушка)
Поля берём **из подписанных секций/заголовков** документа («Název/Označení objektu»,
«Charakteristika…», глава материалов), **НЕ полнотекстовым поиском.** В TZ SO-250 в
геологии есть «mostní objekt» и «lávka SO 222» — полнотекст «most» отравит название/
характеристику и сломает определение типа. Сканируем структуру, а не весь текст.

## 3. Принципы
- **Детерминизм-first:** где документ размечен (секции/заголовки) — regex/структура;
  переиспользовать существующий `analyze_construction_document` (pdfplumber + regex),
  **не дублировать**. LLM (через существующий Vertex-роутинг) — только на мутных/
  неразмеченных кусках.
- **Grounding:** каждое поле с `_source` и confidence; ничего не выдумывать (P26/P29);
  неуверенно → OVĚŘIT/null.
- **Формат выхода = вход рецепта оркестратора** (drop-in замена `request.options`).
- Новая MCP-тулза `extract_tz_fields`: регистрация в server.py + REST-обёртка + manifest
  + CI + `_source`-gate, по 3-слойному принципу.

## 4. Калибровка / акцептация
На SO-250 (zárubní zeď) и SO-202 (most) — у них известная ground-truth из golden-фикстур.

## 5. Kritéria приёмки (продолжить с #94)
**#94** — название + характеристика из TZ SO-250 → «Zárubní zeď…» / «Úhlová železobetonová
zeď», взятые из подписанных секций, **не отравленные** «mostní objekt»/«lávka SO 222» из тела.
**#95** — то же из TZ SO-202 → «Most…» / «Trvalý dálniční most».
**#96** — извлечённые название+характеристика поданы в `detect_object_type` → SO-250
retaining_wall, SO-202 bridge (end-to-end: extract → detect).
**#97** — список элементов извлечён из SO-202 (именованные конструкционные: NK, dříky
pilířů, opěry, úložné prahy, římsy, základy, pilota).
**#98** — класс бетона привязан к каждому элементу (6 пар SO-202: NK C35/45, dříky C35/45,
opěry C30/37, …), без «висящих» классов без элемента.
**#99** — каждое извлечённое поле несёт `_source` (страница/раздел) + confidence; неуверенно
→ OVĚŘIT/null; ничего не выдумано.
**#100** — выход extract по форме = вход рецепта (drop-in вместо `request.options`):
оркестратор прогоняет SO-202 на выходе extract end-to-end → реальный Excel, обобщённые
элементы получают верный контекст. Это и есть приз автономии.

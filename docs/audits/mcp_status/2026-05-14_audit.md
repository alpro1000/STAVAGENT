## 3. OAuth 2.0 pro ChatGPT

✅ **Implementováno**, `routes.py:149-169` (`/api/v1/mcp/oauth/token`):

- Grant type: pouze `client_credentials` (ChatGPT Connectors flow).
- `client_id = API key` → bearer token = ten samý API key (stateless, bez DB lookup na každý MCP call).
- Vrací 400 `unsupported_grant_type` pro jiný grant; 401 při neplatném klíči.
- ⚠️ **Nepotvrzeno end-to-end z ChatGPT klienta.** Code path je `mcp_auth.oauth_token(client_id, client_secret)` v `auth.py` — wiring vypadá kompletní.

## 4. API key validation `sk-stavagent-*`

✅ **Implementováno:**

- Format: `f"sk-stavagent-{secrets.token_hex(24)}"` (`auth.py:130-132`) → 48 hex znaků, prefix `sk-stavagent-`.
- Storage: SQLite `mcp_keys.db` (`auth.py:71`), tabulka `mcp_api_keys` s bcrypt password_hash + credits + `is_active` flag, WAL journal mode pro paralelní čtení.
- Per-thread connection pool (`_db_pool`, `_pool_lock`) — thread-safe.
- Validation extract: `_extract_bearer(authorization)` v `routes.py:373-380` parsuje `Authorization: Bearer <api_key>`.
- Atomic credit deduction: `UPDATE WHERE credits >= cost` (per CLAUDE.md §2 MCP Auth).
- Rate limit: 10 attempts / 60 s per IP (`auth.py:52-66`).
- Free tier (`/tools/otskp`, `/tools/classify`): no auth required — REST wrapper má `authorization` jako Optional bez `_extract_bearer` gate (`routes.py:246-266`).

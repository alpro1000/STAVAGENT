## 5. Souhrn nálezů — pro navazující golden-test / fix task

Seřazeno dle dopadu (vše DESKRIPTIVNÍ, žádný fix zde):

1. **🔴 KROSParser nevytáhne AspeEsticon-XC4 soupis (0/3373).** Root `<XC4>` → `_has_xc4_prices` předběhne `_detect_format`/`ASPE_XC4` větev → 0 položek. Dedikovaný `xc4_parser` (umí 3373/3373) je nedosažitelný. `format_detector` + `SourceFormat` enum AspeEsticon vůbec neznají. **Nejvyšší dopad** — celý soupis SO 202 se v produkční cestě ztratí. *(kros_parser.py:43,57-71,151-160; format_detector.py:77; models.py:18-26)*
2. **🟠 extract_tz_fields: špatný `object_code` (SO 101 ≠ SO 202)** na TZ i statice (první SO match v sekci, chytá referencovaný silniční objekt). *(extract_tz_fields.py:138-146)*
3. **🟠 extract_tz_fields: `object_name`/`charakteristika` = prázdné → `detect_object_type` selže** (neurčí bridge, ač classifier určí `mostní`). *(extract_tz_fields.py:69,149-193; detect_object_type.py)*
4. **🟠 extract_tz_fields: narativní false-positives** (31/37 „elementů" jsou věty). Precision element-stem regexu. *(extract_tz_fields.py:75-86,208-251)*
5. **🟠 Klasifikátor: 23/28 výkresů mislabeled** (mostly RO @0.43) — sparse text + RO keyword bleed; + nekonzistentní cat/sub páry (RO/VY-ARM, TZ/VY-ARM). *(document_classifier.py:109-122,155-161,558-566)*
6. **🟡 Geometrie/technologie NK strukturovaně nepokryta** žádným parserem (§3.5, §4A) — chybějící patterny/pole, ne chyba; to je hlavní budoucí feature-gap (Stage 2 / vision).
7. **🟡 Soupis třída = katalogová mez „DO Cxx/yy"** ≠ skutečná TZ třída; žádné křížové párování TZ↔soupis. *(§3.2)*
8. **🟡 PDFParser na statice** generuje 2 spurious „pozice" ze 154 výpočtových tabulek; na TZ 0/9 (OK). *(pdf_parser.py)*
9. **🟡 format_detector vrací `pdf_tz` pro všechny výkresy** (žádné rozlišení výkres vs TZ). *(format_detector.py:87-103)*
10. **⚪ Regex robustnost:** `concrete_class` greedy `\d{2,3}` → „C225/30" artefakt (4×). *(document.py:22)*

**Doporučení pro golden-test task (NEdělat zde):** golden testy psát PROTI OPRAVENÉMU chování, ne proti current. Pro soupis použít `xc4_parser` ground-truth (3373 / 6 betonových s Δ0 %). Pro TZ testovat materiály (✅ funguje) zvlášť od geometrie (dnes prázdná — buď test „pominuto" jako baseline, nebo gated až po implementaci geometry pole). Nález #1 a #2 jsou pravděpodobně prerekvizity před smysluplnými soupis/TZ goldeny.

# STAVAGENT → Испания (obra civil / licitaciones): план адаптации

**Версия:** 1.1 · **Дата:** 2026-07-02 · **Статус:** черновик стратегии (не обязательство)

> **Changelog v1.1:** уточнено позиционирование по итогам разбора рынка — STAVAGENT
> это **помощник сметчика/přípravář** (пред-расчёт монолита: нормо-часы + опалубка +
> такты + график), НЕ конкурент сметным программам. Ведёт **производственный движок**,
> модуль поиска кодов — вторичен. Лицензия ценника (BEDEC) понижена с критического
> гейта до **опции** (часы берутся из собственных labor-norms, испанский ценник нужен
> только для вывода оценённых позиций). Добавлен §2b — проверенная конкурентная граница
> vs Presto Cost-It / CYPE Arquímedes.

> **English TL;DR:** STAVAGENT is a **work-preparation (přípravář) helper** for the
> estimator — concrete-works pre-planning (norm-hours + formwork system + pour takts +
> schedule), NOT a competitor to Spanish budgeting software (Presto/Arquímedes). The
> calculation engine is ~95% country-agnostic (physics + EN norms); expansion is a
> bounded *adapter* (~4–7 weeks eng), NOT a rewrite. Spanish tools (Cost-It, Arquímedes)
> already do Revit→budget (mediciones + presupuesto + catalog-average hours) but do NOT
> do formwork-system selection, pour takts, or schedule — that gap is STAVAGENT's lane.
> Real cost/risk is a Spain-embedded partner + go-to-market; the price licence (BEDEC)
> is optional. Sequence: finish + secure + monetize Czech FIRST, then Spain via partner.

## 0. Вывод одной страницей

- **Позиционирование: помощник сметчика, не конкурент.** STAVAGENT = přípravář-слой (пред-расчёт монолитных работ: нормо-часы + предварительный выбор опалубки + такты + график). Испанские сметные программы (Presto/Arquímedes) этого не делают — см. §2b.
- **Веди производственным движком, каталог — вторичен.** Ценность = то, чего нет у Cost-It/Arquímedes (система опалубки + такты + график + бригада). Модуль поиска кодов работ — вспомогательный, для Испании наименее важный.
- **Движок универсален.** Реальная связка расчётного ядра с чешским каталогом — ~12 строк кода + валютное именование (`_czk`). Физика (pour/formwork/curing/rebar/schedule/PERT/pump) и нормы (DIN 18218, EN 206/13670/12812) переносятся как есть. Опалубка DOKA/PERI/**ULMA**(исп.)/NOE — панъевропейская.
- **Классификатор спроектирован многоязычным by design:** `type_core` = language-neutral онтология; `dictionaries.<lang>` = языковые словари. Добавить Испанию = добавить `dictionaries.es`.
- **Экспансия = адаптер, не второй продукт.** Чистая инженерия до рабочего MVP: **≈ 4–7 недель**.
- **Настоящее бутылочное горло — НЕ код и НЕ ценник:** партнёр внутри испанского рынка + продажи в гостендерах. Лицензия BEDEC — **опция** (часы из собственных labor-norms; испанский ценник только для оценённых позиций).
- **Ниша:** `obra civil / licitaciones` (дороги, мосты, гостендеры) — зеркало чешской ниши ŘSD/SFDI. НЕ воевать с Budquo/Presto за edificación/reformas.
- **Последовательность жёсткая:** Чехия закрыта (безопасность) → безопасна → **платит** → деньги финансируют партнёра → только тогда Испания.

## 2b. Конкурентная граница (проверено по испанским источникам, 2026-07)

**«Встроено в Revit, сразу в позицию сметы» — правда**, это два продукта:
- **Presto + плагин Cost-It** — двусторонний мост Revit ↔ Presto: тянет `mediciones` (объёмы) из BIM-модели, генерит `presupuesto`, мапит элементы модели в `partidas`.
- **CYPE Arquímedes + Open BIM Quantities** — прямой коннектор Revit → Arquímedes, авто-присвоение позиций через параметр Key Note.

Плюс испанские ценники (**BEDEC / региональные базы**) дают `precios descompuestos` — каждая partida разложена на **труд с часами** (oficial 1ª + ayudante), материал, машины, с `rendimiento` (напр. опалубка стен ~18–22 m²/смену, бригада 2+1).

**Граница — тонкая, не переоценивать:**

| | Presto Cost-It / Arquímedes / BEDEC | STAVAGENT |
|---|---|---|
| Объёмы из модели | ✅ | нет (и не надо) |
| Цена по ценнику | ✅ | вторично |
| Нормо-часы | ✅ **есть**, но каталожное **среднее на единицу** (h/m²) | **проектный расчёт** |
| Выбор **системы опалубки** (Framax vs Frami, давление DIN 18218, высота záběru) | ❌ | ✅ |
| **Такты** (záběry), окно бетонирования, пересменка на мега-заливке | ❌ | ✅ |
| Зрение по классу среды, **график** (Gantt) заливок, аренда опалубки | ❌ | ✅ |

**Вывод:** испанский софт = rozpočtář-слой (сколько и почём). STAVAGENT = **přípravář-слой** (как это строить). Нормо-часы у них есть, но статичное среднее из каталога, а не производственный план конкретной конструкции. Между сметой и стройкой — пусто, и это ниша STAVAGENT.

**Как проверить самому (30 мин):** (1) демо Cost-It / Arquímedes — на выходе mediciones + partidas + precio, нет выбора опалубки/тактов/графика; (2) CYPE generador «encofrado muro» — descompuesto с часами на m² = среднее на единицу; (3) спросить партнёра: *«Cuando preparas un muro de 156 m: ¿el software te dice cuántos tactos, qué sistema de encofrado y el cronograma — o solo el precio por m²?»* → ответ «solo el precio».

**Источники:** espaciobim.com/presto · rib-software.es/cost-it · revit.arquimedes.cype.es · learning.cype.com (interoperabilidad Revit-Arquímedes) · generadordeprecios.info (encofrado muro descompuesto) · diputaciondepalencia.es (base precios descompuestos).

## 4. Фаза 2 — Решение по данным (опция, НЕ гейт)

> ⚠️ **v1.1:** это больше не критический гейт. Ядро ценности (нормо-часы + опалубка +
> такты + график) даётся **без** испанского ценника — часы из собственных
> `labor-norms.ts`, опалубка из своего каталога. Испанский ценник нужен ТОЛЬКО если
> хочешь выводить **оценённые** позиции (€). Значит: MVP можно показать вообще без
> BEDEC/PREOC; ценовой слой — отдельный, поздний, опциональный модуль.

Если/когда нужен вывод в € — развилка:

| | Сценарий A — открытые базы | Сценарий B — лицензия BEDEC |
|---|---|---|
| Данные | PREOC / CYPE generador / IVE / регион. базы | BEDEC (ITeC) полный |
| Покрытие | частичное, разнородное | максимальное, признанное |
| Стоимость данных | ~0 € | лицензия ITeC (€, уточнить) |
| Юр. риск | низкий (открытые/полуоткрытые) | чистый (лицензировано) |
| Скорость старта | быстро | после переговоров/оплаты |
| Для гостендеров | «ориентир» | «признанный ценник» |

**Рекомендация:** стартовать пилот на **A** (доказать движок + BC3 на реальных данных дёшево), лицензию **B** брать, когда чешские деньги её оправдывают и партнёр подтвердил спрос. НЕ платить за BEDEC до пилота.

- [ ] **2.1 Уточнить цену лицензии BEDEC** (ITeC) — точная цифра сейчас неизвестна, поставить как открытый вопрос.
- [ ] **2.2 Проверить лицензии открытых баз** (PREOC/CYPE/региональные) на право программного использования.

## 11. Точки go/no-go

1. **После Фазы 1:** боль подтверждена (≥5 из 10) И есть кандидат в партнёры → иначе пауза, не поражение.
2. **После Фазы 2:** решён вопрос данных — но MVP (производственный план) не блокируется отсутствием ценника; € вывод откладывается, если чистого доступа нет.
3. **Перед Фазой 3:** G0.1–G0.3 (Чехия безопасна + платит) закрыты → иначе стоп.
4. **После Фазы 4 (пилот):** ±10–15% + партнёр верит результату → иначе итерация или стоп.
